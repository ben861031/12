公司通用問卷調查系統 v1.9（Excel 整潔統計報表版）

本版新增：
- Excel 第一張「選項統計總表」改為橫向報表，不再使用重複題目名稱的長清單。
- 每個選項題各自形成獨立區塊，依序顯示「選項、數量、占填寫人數」。
- 尺寸題會直接排列為 XS、S、M、L、XL、2XL 等欄位，最右側顯示合計。
- 新增第二張「部門交叉統計」，以部門為列、選項為欄，並提供各部門小計與最下方總計。
- 第三張「填寫明細」保留每位同仁的完整答案供核對。
- 單選、下拉、部門選單與複選題皆會自動套用相同的整潔報表格式。

v1.8 新增：
- 公司人員模式下，以「問卷 ID＋人員 ID」建立唯一回覆，每位同仁每份問卷只能填寫一次。
- 管理員刪除該筆回覆後，同仁即可重新填寫。
- 管理員開啟既有結果時，舊的隨機回覆文件會安全轉換成唯一識別文件；答案內容不變。
- 新增完整人員管理：新增、修改、啟用／停用、刪除、搜尋、Excel 匯入／匯出與標準範本。
- 與行政部活動調查系統直接共用 Firestore 的 members 集合，所有人員異動即時同步。
- 名單依部門設定順序，再依員工編號排序；前台只顯示啟用人員。
- 匯入格式完全沿用聚餐調查系統：部門、姓名、員工編號、狀態。

重要：請依「Firestore規則_v1.8更新片段.txt」更新 universalResponses 規則，否則防重複與既有資料轉換無法正確運作。

v1.7 新增：
- Excel 匯出改為「統計總表」與「填寫明細」兩個工作表。
- 統計總表固定排列在第一個工作表，開啟檔案即可先看到摘要。
- 顯示問卷名稱、匯出時間與總填寫份數。
- 自動彙整部門分布，以及單選、下拉、部門選單、複選題的各選項數量與比例。
- 例如衣服尺寸調查會直接列出 XS、S、M、L、XL 等尺寸各有多少人選擇。
- 第二個工作表保留每位同仁的完整填寫明細。

v1.6 新增：
- 統計總填寫人數、填寫部門數與最近填寫時間。
- 部門分布、單選及下拉題自動產生人數、百分比與圓餅圖。
- 複選題顯示各選項被選人數、比例與進度條。
- 簡答與長文題整理為具名文字回覆清單。
- 每筆填寫結果新增「編輯」及「刪除」操作。
- 管理員可修正填寫者部門、姓名、員工編號及全部答案。
- 刪除時顯示姓名、員編並要求兩次確認；刪除後可讓同仁重新填寫。

重要：請依「Firestore規則_v1.6更新片段.txt」更新 universalResponses 規則，否則編輯功能會被 Firebase 拒絕。

v1.5 調整：
- 前台不再自動替題目加上 1、2、3 等題號。
- 題目名稱完全依照後台輸入內容顯示；需要編號時可自行在題目前輸入「1.」。
- 「圖片／說明」題型改為先顯示題目名稱與說明，再顯示參考圖片。
- 所有含圖片的題目均統一為「題目名稱 → 說明 → 圖片 → 作答欄位」。

沿用 v1.4 Google Drive 圖片網址功能，不需 Firebase Storage。

v1.4 新增：
- 不需要啟用或付費使用 Firebase Storage。
- 頁首圖片及每一題的參考圖片可直接貼 Google Drive 分享網址。
- 系統會自動辨識常見 Drive 分享網址並轉成圖片顯示網址。
- 後台輸入網址時會立即顯示預覽，權限不正確時會顯示提醒。
- 一般 https 圖片網址與既有 assets 相對路徑仍可使用。

Google Drive 圖片使用方式：
1. 將圖片上傳至 Google Drive。
2. 在圖片上按右鍵 → 共用／分享。
3. 一般存取權改為「知道連結的使用者」。
4. 權限選擇「檢視者」。
5. 複製分享網址，貼到後台的「頁首參考圖片網址」或題目的「參考圖片網址」。
6. 確認後台已顯示圖片預覽，再儲存問卷。

部署時請連同 css/v1.4.css 一起上傳；不需要設定 Storage Rules。

v1.2 原有功能：
- 可為每份問卷選擇「使用公司人員資料庫」或不收集身分。
- 後台可勾選本次開放填寫的部門；未勾選代表全部部門。
- 前台依部門篩選啟用人員，選擇姓名後自動帶入員工編號。
- 回覆及 Excel 結果會記錄部門、姓名、員編與人員 ID。
- 外套尺寸範本改用統一身分區塊，不再要求手動輸入姓名與員編。

本版修正：
- 修正 config.js 已存在卻被誤判為「尚未設定 Firebase」。
- 修正 Firebase 尚未初始化時點擊登入造成 signInWithPopup 錯誤。
- 更新 CSS、config.js 與 app.js 快取版本，避免 GitHub Pages 沿用舊檔。

用途：
- 取代一般 Google 表單，建立公司內部各類問卷。
- 與行政部活動調查系統完全分開。
- Firebase 集合使用 universalForms、universalResponses。

目前功能：
- 建立、編輯、刪除多份問卷。
- 每份問卷具備獨立前台網址。
- 題型：簡答、長文、單選、複選、下拉選單、部門選單、圖片／說明。
- 設定必填、截止時間、草稿／開放／關閉狀態。
- 題目上移、下移及刪除。
- 公開前台填寫與 Firebase 儲存。
- 後台查看結果與 Excel 匯出。
- 一鍵建立「立領撞色外套尺寸調查」範本。

檔案結構：
- index.html：前台與後台頁面結構
- css/app.css：介面樣式
- js/config.js：Firebase 設定
- js/app.js：問卷建立器、填寫與結果功能
- assets/company-logo.png：公司識別
- assets/uniform-survey-reference.jpg：原 Google 問卷參考截圖
- Firestore規則新增區塊.txt：需要合併到共用 Rules 的新集合規則

部署順序：
1. 請建立新的 GitHub Pages 儲存庫或獨立資料夾，不要覆蓋聚餐系統。
2. 上傳本資料夾中的 index.html、css、js、assets。
3. 將「Firestore規則新增區塊.txt」中的兩個 match 區塊加入現有 Rules，放在最後拒絕規則之前。
4. 以管理員 Google 帳號登入後台。
5. 按「建立外套尺寸範本」，補上截止時間並將狀態改為「開放填寫」。
6. 將參考圖片上傳至 Google Drive、開啟連結檢視權限，再將分享網址貼入題目的「參考圖片網址」。

後台網址：index.html#admin
前台網址：由後台「複製前台網址」取得，格式為 index.html#form/問卷ID
